<task type="auto">
  <name>Task 1: Install Tailwind via Astro integration</name>
  <files>package.json, astro.config.mjs, src/styles/global.css</files>
  <action>
    Run the official Astro integration command:
    ```
    npx astro add tailwind --yes
    ```
    This installs the required packages and updates `astro.config.mjs` automatically.

    After installation, check what version of Tailwind was installed:
    - If Tailwind v3: `global.css` will have `@tailwind base; @tailwind components; @tailwind utilities;` directives added. Verify they were added — if not, add them at the top of `global.css` above the existing `:root` block.
    - If Tailwind v4: the integration uses a Vite plugin; directives are handled differently. Check that `@import "tailwindcss"` or equivalent is present.

    In either case: preserve the existing `:root { ... }` block and all CSS var definitions in `global.css` — do not remove them.
  </action>
  <verify>npm run build exits 0</verify>
  <done>AC-1 satisfied: build passes with Tailwind integrated</done>
</task>

<task type="auto">
  <name>Task 2: Map design tokens to Tailwind theme</name>
  <files>tailwind.config.mjs</files>
  <action>
    Create or update `tailwind.config.mjs` to extend the Tailwind theme with the project's design tokens. Target Tailwind v3 config format; adapt if v4 was installed (v4 uses CSS `@theme` blocks instead).

    **For Tailwind v3** — set `tailwind.config.mjs` to:
    ```js
    /** @type {import('tailwindcss').Config} */
    export default {
      content: ['./src/**/*.{astro,html,js,jsx,ts,tsx}'],
      theme: {
        extend: {
          colors: {
            paper: '#eceef1',
            'paper-2': '#e1e4e8',
            ink: '#141416',
            'ink-2': '#2a2a2d',
            'ink-muted': '#6b6b72',
            accent: '#d94a28',
          },
          fontFamily: {
            body: ['Inter', 'system-ui', 'sans-serif'],
            mono: ['"JetBrains Mono"', 'ui-monospace', 'monospace'],
            serif: ['"IBM Plex Serif"', '"Fraunces"', 'Georgia', 'serif'],
          },
          maxWidth: {
            doc: '1200px',
          },
          screens: {
            '3xl': '1920px',
          },
        },
      },
      plugins: [],
    }
    ```

    **For Tailwind v4** — add the equivalent token mappings inside a `@theme` block in `global.css`:
    ```css
    @theme {
      --color-paper: #eceef1;
      --color-paper-2: #e1e4e8;
      --color-ink: #141416;
      --color-ink-2: #2a2a2d;
      --color-ink-muted: #6b6b72;
      --color-accent: #d94a28;
      --font-body: 'Inter', system-ui, sans-serif;
      --font-mono: 'JetBrains Mono', ui-monospace, monospace;
      --font-serif: 'IBM Plex Serif', 'Fraunces', Georgia, serif;
      --breakpoint-3xl: 1920px;
    }
    ```
    In v4, the `@theme` block should sit after `@import "tailwindcss"` and before the existing `:root` block.

    Do not remove the existing `:root` CSS var definitions — they are still used by components with scoped `<style>` blocks.
  </action>
  <verify>
    Run: npm run build
    Check: dist/index.html is generated without errors.
    Spot check: add a temporary `class="text-accent"` to any element in index.astro, build, grep dist/index.html for the accent color hex (#d94a28) — it should appear in the generated CSS.
    Remove the temporary class after verifying.
  </verify>
  <done>AC-2 and AC-3 satisfied: design tokens resolve as Tailwind classes; CSS vars still present in output</done>
</task>
