<objective>
## Goal
Make the Work, Experience, and Contact sections fully responsive across mobile (375px), tablet (768px), desktop (1280px), and 4K (1920px+) — converting fixed grid layouts to Tailwind responsive classes and adding fluid type where needed.

## Purpose
The above-the-fold (Nav + Hero + wrapper) is responsive from 03-02. The remaining content sections still use fixed-column grids in scoped CSS with minimal or missing mobile media queries. This closes the responsive gap for the full page.

## Output
Three updated Astro components with Tailwind-driven responsive grids, scoped CSS retained only for design tokens/non-layout styles.
</objective>

## AC-1: Work section grid responsive
```gherkin
Given the Work section renders work case cards
When the viewport is < 768px (mobile)
Then cards stack in a single column (index, title, body, tags vertically)
When the viewport is ≥ 768px (desktop)
Then cards show the 3-column grid (72px index | content | 260px tags)
When the viewport is ≥ 1920px (4K)

```

## AC-2: Experience timeline responsive
```gherkin
Given the Experience timeline renders year | entry pairs
When the viewport is < 768px (mobile)
Then year and entry stack vertically (year label above entry text, no double border)
When the viewport is ≥ 768px (desktop)
Then timeline is the 2-column grid (120px year | entry)
```

## AC-3: Contact footer responsive
```gherkin
Given the Contact footer renders 3 columns (Stack, Languages/Base, Get in touch)
When the viewport is < 640px (mobile)
Then columns stack to a single column
When the viewport is between 640px and 767px (tablet)
Then columns render as 2-column (first two columns | third wraps below)
When the viewport is ≥ 768px (desktop)
Then footer shows 3 columns at full width
```

<task type="auto">
  <name>Task 1: Work section — Tailwind grid + fluid h3</name>
  <files>src/components/Work.astro</files>
  <action>
    1. On `<article class="doc-case">`, add Tailwind classes to drive the grid:
       `grid gap-3 md:grid-cols-[72px_1fr_260px] md:gap-9`
       — `grid` makes it a grid container at all sizes (single-col default on mobile)
       — `gap-3` = 12px mobile gap; `md:gap-9` = 36px desktop gap
       — `md:grid-cols-[72px_1fr_260px]` = 3-col at md+

    2. On `<div class="stack">`, add `md:justify-end` (already has `justify-content: flex-end` in scoped CSS — move that to Tailwind):
       `class="stack md:justify-end"`

    3. In scoped CSS, remove from `.doc-case`:
       `display: grid;`
       `grid-template-columns: 72px 1fr 260px;`
       `gap: 36px;`
       Also remove the `@media (max-width: 767px)` block for `.doc-case` grid override and `.stack justify-content`.

    4. In scoped CSS, update `.doc-case h3` font-size:
       Replace `font-size: 30px` with `font-size: clamp(22px, 2.5vw, 30px)`
       — Scales down at narrow viewports, stays at 30px on desktop, grows slightly on 4K.

    Avoid: touching colors, fonts, letter-spacing, border, or other non-layout scoped styles.
    Avoid: adding Tailwind utilities for padding/border/font-family — those stay in scoped CSS.
  </action>
  <verify>
    Build passes: `npm run build` (or `npx astro build`) exits 0.
    Dev server: at 375px viewport, work cards stack 1-col (index above title above body above tags).
    At 1280px, 3-col grid is intact with tags right-aligned.
  </verify>
  <done>AC-1 satisfied: Work grid collapses on mobile, 3-col on desktop, h3 clamp applied</done>
