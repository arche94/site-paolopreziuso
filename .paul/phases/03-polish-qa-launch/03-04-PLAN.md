<task type="auto">
  <name>Task 1: Add SEO meta tags to Base.astro</name>
  <files>src/layouts/Base.astro, src/data/site.json</files>
  <action>
    1. In `src/data/site.json`, add two fields at the top level:
       - `"description"`: a concise 1-sentence meta description (≤155 chars) summarising Paolo's professional value — derive from the existing `hero.lede`
       - `"siteUrl"`: `"https://paolopreziuso.com"`

    2. In `src/layouts/Base.astro`:
       - Extend Props interface: keep `title: string`; add `description?: string` (optional), `canonicalUrl?: string` (optional)
       - Import `site` from `../data/site` and use `site.description` and `site.siteUrl` as defaults when props are not passed
       - Add to `<head>` after the existing `<title>` tag, in this order:
         ```html
         <meta name="description" content={description ?? site.description} />
         <meta property="og:type" content="website" />
         <meta property="og:url" content={canonicalUrl ?? site.siteUrl} />
         <meta property="og:title" content={title} />
         <meta property="og:description" content={description ?? site.description} />
         <meta name="twitter:card" content="summary" />
         <meta name="twitter:title" content={title} />
         <meta name="twitter:description" content={description ?? site.description} />
         ```

    Avoid: adding og:image — no suitable image asset exists yet; omit rather than point to SVG logo.
    Avoid: changing the existing Props interface in a breaking way — `title` must remain required.
  </action>
  <verify>npm run build && grep -c "og:title" dist/index.html (expect 1)</verify>
  <done>AC-1 satisfied: build output contains description + OG + Twitter Card tags with non-empty values</done>
</task>
