<task type="auto">
  <name>Task 1: Create GitHub Actions deploy workflow</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
    Create `.github/workflows/deploy.yml`:

    **Trigger:** `on: push: branches: [main]`

    **Job: build-test-deploy** (runs-on: ubuntu-latest)

    Steps:
    1. `actions/checkout@v4`
    2. `actions/setup-node@v4` with `node-version: '20'`, `cache: 'npm'`
    3. `npm ci`
    4. `npm run build` — verify static output in `dist/`
    5. `npx playwright install --with-deps chromium` — install browser in CI
    6. `npx playwright test` — smoke test against dev server (webServer config handles startup)
    7. Deploy: `npx wrangler pages deploy dist/ --project-name=${{ secrets.CLOUDFLARE_PROJECT_NAME }}`
       - env: `CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}`
       - env: `CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}`

    **Required secrets (documented in workflow comments):**
    - `CLOUDFLARE_API_TOKEN` — Cloudflare API token with Cloudflare Pages edit permission
    - `CLOUDFLARE_ACCOUNT_ID` — found in Cloudflare dashboard right sidebar
    - `CLOUDFLARE_PROJECT_NAME` — the name given to the project in Cloudflare Pages

    Add a comment block at the top of the workflow file explaining the three secrets needed and where to find them.

    Avoid: Adding `wrangler` as a project dependency — `npx wrangler` is sufficient and avoids version pinning maintenance.
    Avoid: Caching Playwright browsers in this phase — adds complexity without meaningful time savings for a 2-test suite.
  </action>
  <verify>Run `cat .github/workflows/deploy.yml` and confirm all steps are present; no YAML syntax errors (validate with `npx js-yaml .github/workflows/deploy.yml` or similar)</verify>
  <done>AC-1 satisfied: workflow file exists with correct structure</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <what-to-do>
    Create the Cloudflare Pages project and add GitHub secrets.

    **Step 1 — Create Cloudflare Pages project:**
    1. Go to Cloudflare Dashboard → Workers & Pages → Create → Pages
    2. Connect to Git → select the `site-paolopreziuso` repo
    3. Set build command: `npm run build`
    4. Set output directory: `dist`
    5. Note the project name you give it (e.g. `site-paolopreziuso`)

    **Step 2 — Get credentials:**
    - Account ID: visible in Cloudflare dashboard right sidebar (any page)
    - API Token: My Profile → API Tokens → Create Token → use "Edit Cloudflare Workers" template, scope to your account

    **Step 3 — Add GitHub secrets:**
    Go to the GitHub repo → Settings → Secrets and variables → Actions → New repository secret:
    - `CLOUDFLARE_API_TOKEN` → paste the token
    - `CLOUDFLARE_ACCOUNT_ID` → paste the account ID
    - `CLOUDFLARE_PROJECT_NAME` → the project name from Step 1

    Note: You can disable the auto-deploy from Cloudflare's git integration (created in Step 1) since GitHub Actions will handle deploys — or leave it; both work.
  </what-to-do>
  <resume-signal>Type "secrets configured" when all three GitHub secrets are added</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <what-to-do>
    Trigger the first deploy and wire the custom domain.

    **Step 1 — Trigger deploy:**
    Push any commit to `main` (e.g. update a comment in index.astro). Watch the Actions tab in GitHub — confirm the workflow passes all steps.

    **Step 2 — Verify at Cloudflare URL:**
    In Cloudflare Dashboard → Pages → your project → visit the `*.pages.dev` preview URL.
    Confirm: placeholder page loads, no errors.

    **Step 3 — Add custom domain:**
    In Cloudflare Dashboard → Pages → your project → Custom domains → Add domain → enter `paolopreziuso.com`
    Follow prompts (if domain is already on Cloudflare, DNS updates automatically; if not, add the CNAME record at your registrar).

    **Step 4 — Verify domain:**
    Wait for DNS propagation (minutes to hours). Visit https://paolopreziuso.com — confirm it loads the placeholder page.
  </what-to-do>
  <resume-signal>Type "domain live" when https://paolopreziuso.com serves the site, or describe any issues</resume-signal>
</task>
